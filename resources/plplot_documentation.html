The Current PLplot Documentation Project

The previous form of the PLplot documentation was quite extensive but had two issues. (1) It was written in LaTeXinfo form which is no longer supported. (2) It was dated (no documentation updates since 1995 because of issue [1]). To address issue (1) Rafael Laboissiere with some help from Alan W. Irwin converted the existing documentation from LaTeXinfo to DocBook form. Thus, all chapters of the original documentation are now available in html form as well as DVI, and postscript form. Issue (2) is being addressed by Alan W. Irwin with some help from Rafael Laboissiere, and others are encouraged to participate as well. The DocBook source of the documentation is now under cvs control at the PLplot sourceforge project site. To build the documentation, follow the cvs instructions at that site to synchronize your local plplot directory tree with the cvs repository (using cvs checkout or cvs update as appropriate). Then follow the directions in plplot/doc/docbook-sgml/README.developers to build the dvi, postscript, and html results. If you do not have cvs write access, please post your suggested documentation changes to Alan W. Irwin as a patch file on one of the *.sgml docbook source files. The sgml syntax for DocBook is straightforward and can easily be inferred from the existing sgml files, but there is excellent documentation of DocBook as well, if you really need it.

Motivation for using DocBook

It is well documented! ;-)

Docbook fulfills documentation needs using industry-standard DTD's (Document Type Definition's) based on SGML (Structured Markup Language). The idea is that the documenter uses logical markup ("this is a section head"), and the typesetting details (bold face or whatever for section heads, etc.) are taken care of by the DTD's. This important and powerful distinction between logical markup and typesetting allows documenters to concentrate on producing documentation while the look and feel of the results can be adjusted afterward by adjusting the DTD's through so-called driver files.

DocBook is gaining a lot of momentum now: O'Reilly uses it to compose their books, the Linux Documentation Project uses it (see this), the KDE project uses it (see this), the Gnome project uses (see this), and the latest version of SGML-Tools is based on DocBook. Another big project that will use DocBook for its documentation is the General Graphics Interface (GGI) Project.

DocBook is just a DT definition. The jade suite or the OpenJade suite can be used to convert from the SGML sources using several different backends (like HTML and LaTeX). There are some tools available for producing nroff and TeXinfo outputs from DocBook sources: docbook2X (in Perl) and dbtotexinfo (in Java).

There are at least two nice aspects to using DocBook. First, the source is suitable for CVS control (see above). Second, it is possible to parse the source with existing SGML parsers (e.g. in Perl). Thus, for example, it is possible to automatically extract all PLplot function definitions.
